# METRICS.md: Engineering & Growth Analytics

## 1. The Single North Star Metric
Our unambiguous North Star Metric is **Total Annualized Capital Recovery Identified (TACRI)**. 

### Why this metric matters:
Unlike traditional software applications that measure success via daily active users (DAU), StackAudit is an episodic diagnostic utility designed for high-impact, periodic evaluations. Measuring simple traffic volume is a misleading vanity vector. Tracking the aggregate dollar amount of software waste our platform surfaces directly quantifies the financial value we expose for technical leads, directly driving user motivation to book a Credex consultation.

---

## 2. Core Operational Input Metrics
To consistently scale our North Star Metric, we actively monitor three critical upstream input metrics:

* **High-Value Math State Trigger Rate:** The percentage of unique browser visitors who input a combination of values that unlocks an identified annualized capital recovery exceeding $500. This tracks our alignment with our target Ideal Customer Profile (ICP).
* **Workspace Interaction Depth:** The average number of tools added to the "Audit Inputs" container per user session. A user stacking 3+ tools demonstrates deep engagement and yields higher audit validity.
* **Frictionless Lead Funnel Conversion Velocity:** The percentage of users who interact with the "Join List" notification element after discovering positive optimization savings.

---

## 3. Immediate Instrumentation Priorities
Upon initial launch, the system will prioritize instrumenting lightweight, non-invasive telemetry hooks using privacy-compliant tools like **Plausible Analytics**:
* Tracking custom page events whenever the `Add Tool` action button is triggered.
* Capturing submission events on the `Join List` button element.
* Measuring layout viewport tracking to confirm that users are successfully scrolling down to read their custom per-tool analysis blocks after updating team parameters.

---

## 4. The Quantitative Pivot Metric
We have established a strict baseline pivot threshold: If **Workspace Interaction Depth drops below 1.2 tools per session over a sample size of 500 unique platform visitors**, it indicates that technical managers find manual input configuration too tedious.

### The Pivot Action Plan:
If this threshold is breached, we will immediately pivot away from manual input arrays and re-engineer the platform's entry point to support a direct, automated copy-paste interface that parses flat `.csv` billing files exported from common startup credit cards.
